# 2. 見落としがちなポイント

## 2-1. データが空の場合を確認する

- お知らせデータがない場合、どう表示させておくの？
- 商品が入っていない時のカートページの状態は？

などなど...
データが無い時の表示が不完全だとユーザに不信感を与えてしまいます。
デザイン時点で決めておけば安心ですね。

## 2-2. 入力できる場所は長いテキストを入れてみる

「お知らせのタイトルに長いテキストを入れたら崩れた」

CMSから入力する場所などによくありがちなミスです。
固定の情報では無い、可変的な実装部分は確認すべきです。

- 逆に短いテキストの時
- そもそも入力がされていない時(空の状態)

合わせて確認できるとより良いですね。

## 2-3. 画像にalt属性が入っているか確認する

**「後でまとめて入れよう」はやめましょう。**
多分忘れます。

- 何らかの問題で画像が表示されなくても代替テキストが表示される
- スクリーンリーダーで読み上げられるので視覚的に理解できないユーザにも優しい
- 検索エンジンに正しく画像の内容を伝えられる

SEOやアクセシビリティの観点からも、alt属性はかなり重要な情報です。
適当に入れずにしっかり考えて入れましょう...

空にする場合もありますが、説明は割愛します。
参考：[HTMLImageElement: alt プロパティ > 装飾的な画像](https://developer.mozilla.org/ja/docs/Web/API/HTMLImageElement/alt#%E8%A3%85%E9%A3%BE%E7%9A%84%E3%81%AA%E7%94%BB%E5%83%8F)

```
<img src="hello.jpg" alt="">
```

# 3. UI Stackのご紹介

**UI Stack**とはアメリカのデザイナーScott Hurffさんが提唱した**UIの考慮すべき5つの状態**という考え方です。UI Stackを意識することで、どの状態でもユーザが迷わず行動できるUIを設計することができます。

この考え方を踏襲すれば「あ、考えてませんでした...修正しておきます...(泣)」という事故は格段に減るのかなと思います。(指摘された時心臓無くなるかと思いました)

1. Ideal State（理想ステート）
2. Empty State（エンプティステート）
3. Error State（エラーステート）
4. Partial State（パーシャルステート）
5. Loading State（ローディングステート）

以上が5つの状態です。
説明すると長くなりそうなのでこの記事では割愛します。

公式リファレンス：[How to fix a bad user interface](https://www.scotthurff.com/posts/why-your-user-interface-is-awkward-youre-ignoring-the-ui-stack/)
個人的に好きな記事：[UI Stackに基づいた状態変化に強いUIを作ろう！](https://digitalidentity.co.jp/blog/creative/ui-stack.html)
