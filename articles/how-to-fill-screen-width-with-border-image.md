# 2. border-imageとは

軽くborder-imageの説明を記載しておきます。
border-imageは、枠線に画像をつけることのできるプロパティです。
以下のような書き方で指定を行います。

```CSS
border-image: [ソース] [スライス] / [幅] / [アウトセット] [リピート];
```

| 記述(プロパティ) | 説明                                                                                                                                   |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| ソース           | 背景画像(色も可)を指定できます。linear-gradientの指定なら通常は2色以上指定しますが、単色で指定するとその色一色の画像として扱われます。 |
| スライス         | 指定した画像を9つの領域に分割することができ、分割された各領域が最終的な枠線の各領域に割り当てられるようになります。                    |
| 幅               | 背景画像(色)の幅を決めます。                                                                                                           |
| アウトセット     | 指定した画像を9つの領域に分割することができ、分割された各領域が最終的な枠線の各領域に割り当てられるようになります。                    |
| リピート(任意)   | 画像の辺領域での表示方法を引き伸ばす・繰り返すなどの指定ができます。                                                                   |

※「ソース」「スライス」「幅」の値の間には「/（スラッシュ）」が必要です。

本来のborder-imageの使い方は以下を参考にしてください。
詳しい説明は本記事では割愛します。

MDN：[border-image](https://developer.mozilla.org/ja/docs/Web/CSS/Reference/Properties/border-image)


## 3-2. やり方

```CSS
border-image:linear-gradient( #bfffbd ) fill 0 //  0 100vw;
```

| 記述                       | 説明                                                                                                                                                  |
| -------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| linear-gradient( #bfffbd ) | ソースに当たる部分です。背景色(画像)をここで指定します。( #000,#fff )のようにカラーコードを2つ記載すればグラデーションになります。                    |
| fill                       | スライスに当たる部分です。border-image は境界線だけに適用されますが、このキーワードを入れることで要素のコンテンツ背景領域までその色で塗りつぶします。 |
| 0                          | 幅に当たる部分です。背景色(画像)の「どこで切るか」を指定します。０にすることで背景色(画像)を分割せず、一つの塊として扱っています。                    |
| //                         | 「幅（width）」を省略して、直接「アウトセット（outset）」を指定することを意味します。                                                                 |
| 0 100vw                    | 「要素の外側にどれだけはみ出させるか」を決めます。**0**で上下方向にはみ出しなし。**100vw**で左右方向に100vw（画面幅いっぱい）にはみ出させます。       |

## 3-3. メリットとデメリット

border-imageで背景色を親要素からはみ出して画面幅いっぱいに広げる実装のメリットとデメリットについてお話します。

### 3-3-1. メリット

- コンテンツはinnerの中に納まってくれる
- HTMLがシンプルに保てる
  - 背景を広げるための親要素を増やす必要が無い
  - CSSのみで解決できる
- z-indexの調整がいらない
  - 擬似要素を使って背景を作るとz-index を調整する手間が発生する

### 3-3-2. デメリット

- コードの可読性が低い
  - 擬似要素を使って背景を作るとz-index を調整する手間が発生する
- 要素自体のborderが使えなくなる
  - 通常のborderが引くことができなくなる
  - 枠線が必要な場合は内側の要素でつけるなどの工夫が必要
